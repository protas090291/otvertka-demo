# Быстрое исправление проблемы с прогрессом задач

## Проблема
Не удается обновить прогресс задач - выдает ошибку.

## Причина
Отсутствуют RLS политики для таблицы `tasks` в Supabase.

## Решение (5 минут)

### 1. Откройте Supabase Dashboard
- Перейдите на https://supabase.com/dashboard
- Выберите ваш проект
- Откройте "SQL Editor"

### 2. Выполните этот SQL скрипт:

```sql
-- Включаем RLS для tasks
ALTER TABLE tasks ENABLE ROW LEVEL SECURITY;

-- Удаляем старые политики
DROP POLICY IF EXISTS "Allow all operations on tasks" ON tasks;
DROP POLICY IF EXISTS "Enable read access for all users" ON tasks;
DROP POLICY IF EXISTS "Enable insert for all users" ON tasks;
DROP POLICY IF EXISTS "Enable update for all users" ON tasks;
DROP POLICY IF EXISTS "Enable delete for all users" ON tasks;

-- Создаем новые политики
CREATE POLICY "Enable read access for all users" ON tasks
    FOR SELECT USING (true);

CREATE POLICY "Enable insert for all users" ON tasks
    FOR INSERT WITH CHECK (true);

CREATE POLICY "Enable update for all users" ON tasks
    FOR UPDATE USING (true);

CREATE POLICY "Enable delete for all users" ON tasks
    FOR DELETE USING (true);
```

### 3. Проверьте результат
- Откройте приложение в браузере
- Попробуйте изменить прогресс задачи
- Ошибка должна исчезнуть

## Тестирование
Откройте файл `test_tasks_api.html` в браузере для тестирования API задач.

## Готово! ✅
После выполнения SQL скрипта обновление прогресса задач будет работать корректно.
